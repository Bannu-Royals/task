Task: Build a Simple Admin & Student Dashboard System

Objective: Design and implement a web application that provides an Admin Dashboard for managing announcements and viewing student statistics, and a Student Dashboard for registered students to view announcements.

Functional Requirements (Admin)

  • Display the total number of registered students.
  • Provide functionality to create, edit, and delete announcements.
  • Each announcement should include: Title, Description/Message, and an auto-generated Date.

Functional Requirements (Student)

  • Students must be able to register/login.
  • Display the list of admin-created announcements (latest first).
  • User-friendly interface with clear navigation.
  • Responsive design for all device sizes.

Expected Features & Flow

  1. At least two roles: Admin and Student.
  2. Students register/login to access their dashboard.
  3. Admin can add, edit, and delete announcements; updates reflect instantly for students.
  4. Store data in any DB (MySQL, MongoDB, Firebase, etc.).
  5. Keep the frontend simple and user-friendly.

Technical Expectations

Any tech stack is acceptable (MERN, Django, Spring Boot, PHP...). Focus on authentication, CRUD operations for announcements, and separate admin/student dashboards.

Suggested UI Elements

Total Registered Students: X

Announcements list with Edit / Delete controls and a form to create new announcements.

Deliverables

  • Working application with separate Admin and Student dashboards.
  • Source code repository (GitHub/GitLab).
  • README with setup instructions.
  • Live demo (if possible).
